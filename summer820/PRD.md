# 콘텐츠 자동화 파이프라인 — 블로그 발행 에이전트

## 문제
> 마케터가 여행/라이프스타일 블로그 콘텐츠를 수작업으로 리서치-집필-발행하는 데 과도한 시간이 소요되는 문제를 AI 에이전트 파이프라인으로 해결한다

- **현재 상태**: 블로그 1건 발행에 리서치 40분 + 집필 60분 + 편집 30분 = 약 2시간 소요. 주 1~2회 발행이 한계
- **원하는 상태**: Slack에서 주제만 입력하면 리서치→초고→카피→디자인 가이드→Notion 저장→블로그 미리보기까지 자동 완료. 주 3~4회 발행
- **성공 기준**: 콘텐츠 1건당 소요 시간 2시간 → 30분 이내, 주간 발행 빈도 2회 → 4회

## 변화 기록
- **Day 1**: "Claude Code가 뭔지도 모르겠고, 터미널이 낯설다" → Claude Code 기본 사용법 익힘
- **Day 6**: "Slack에서 주제만 던지면 5개 에이전트가 협업해서 블로그 초고를 완성하고 Notion에 저장한다" → 콘텐츠 파이프라인 전체 흐름 구축 완료
- **가장 크게 달라진 점**: '도구를 배우는 사람'에서 '도구를 만드는 사람'으로. 스킬 6개를 직접 설계·구현하며 "AI를 쓰는 것"이 아니라 "AI 에이전트를 설계하는 것"의 차이를 체감
